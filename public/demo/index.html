Step 3

六大规则引擎

不需要配置文件，纯 UI 操作，点击即生效

📁 Map Local

将匹配到的请求响应替换为本地文件内容，常用于返回本地 mock 数据

api.example.com/v2/* → /Users/beta/mock/response.json
🌐 Map Remote

将请求转发到另一个地址，常用于切换测试环境 / 前端独立调试

api.example.com/v2/* → staging-api.example.com/v2/*
🏷️ Rewrite Header

修改请求或响应头，可添加、删除或替换 Header 的值

添加 Authorization / 修改 Content-Type / 注入调试 Header
✏️ Rewrite Body

修改请求或响应的正文内容，支持 JSON、正则、纯文本替换

"plan": "free" → "plan": "pro"（测试高级功能）
🐌 Throttling

限制网络带宽，模拟 3G / 4G / 高延迟等弱网条件

限速 500Kbps，延迟 200ms（模拟移动网络）
🚫 Block Request

直接拦截并阻断匹配的请求，可模拟接口不可用、第三方服务挂掉

block cdn.example.com/ads/*（屏蔽广告请求）
规则详解 · 1/6
📁
Map Local
返回本地文件内容，不走真实网络

是什么

指定一个 URL 匹配规则，当请求匹配时，RelayCraft 直接从本地文件读取响应内容返回，完全不发出真实网络请求。

什么时候用

  • 后端接口还没写好，前端需要先跑起来调试 UI
  • 第三方付费 API 想本地 mock 避免每次调用扣钱
  • 想复现特定响应（如错误码、边界数据），保存成文件反复使用
  • 演示 / 截图 时不想依赖网络

⚙️ 创建规则

规则详解 · 2/6
🌐
Map Remote
把请求转发到另一个 URL，不改前端代码

是什么

指定匹配规则和目标地址，请求发出时被拦截并重定向到目标地址。真实发出去，但发到不同的地方。

什么时候用

  • 前端联调：后端本地跑在 localhost:3000，一键把线上域名转发过去
  • 切测试环境：线上出问题，改一条规则切到 staging，不需要改代码 / 重启
  • 调用本地 AI：把请求转发到本地运行的模型服务（如 ollama）
  • 跨域调试：本地起一个中转服务，统一处理 CORS

⚙️ 创建规则

规则详解 · 3/6
🏷️
Rewrite Header
添加、删除或修改请求 / 响应头中的任意字段

是什么

在请求发出去之前或响应到达浏览器之前，动态修改 HTTP Header。可以添加原本没有的 Header，或覆盖已有的值。

什么时候用

  • 注入 Token：后端接口需要 Authorization，但前端漏带了，快速补上调试
  • 修改 UA：模拟不同浏览器或 App 的请求头
  • 绕过单点登录：在测试环境注入已有的登录 Cookie / Session
  • 调试缓存：修改 Cache-Control 或 Etag 强制刷新或命中缓存
  • CORS 调试：添加或覆盖 Access-Control-* 系列 Header

⚙️ 创建规则

规则详解 · 4/6
✏️
Rewrite Body
修改请求或响应的正文内容，支持 JSON / 正则 / 纯文本

是什么

拦截并修改 HTTP Body 的内容。JSON 数据可以直接用路径定位字段替换，也可以用正则匹配任意文本。

什么时候用

  • 测试付费功能：把 "plan": "free" 改成 "plan": "pro"，看前端 UI 变化
  • 复现错误响应：把 code 改成 500，看前端错误处理是否正确
  • 修改接口数据：长列表数据太多，改成只有 2 条，方便 UI 测试滚动和分页
  • 注入测试数据：给响应 JSON 加一个特殊的 error_code，前端有没有做判断

⚙️ 创建规则

规则详解 · 5-6/6

Throttling · Block Request

🐌
Throttling
模拟网络限速
在真正发出请求前加入人为延迟和带宽限制，模拟真实网络环境。
带宽: 500 Kbps  |  延迟: 200 ms
使用场景：
→ 验证 App 在弱网下加载状态是否友好
→ 测试超时逻辑和重试机制是否正常
→ 确认骨架屏和渐进式加载的实现效果
🚫
Block Request
直接拦截请求
匹配到的请求直接被阻断，不发出网络请求，也不会有响应返回。
block cdn.example.com/ads/*
使用场景：
→ 屏蔽页面广告请求，加快加载速度
→ 禁用特定 SDK / 监控脚本，方便本地调试
→ 模拟第三方接口不可用，测试前端容错逻辑

Step 5

MCP 连接外部 AI

把 RelayCraft 的流量数据暴露给 Cursor / Claude Desktop，AI 帮你分析

🔗 Cursor

1
Cursor → Settings → Models → MCP Servers
2
添加 MCP Server，填入 RelayCraft 提供的地址
3
在 Cursor 直接问："帮我看看最近的 API 请求有没有问题"
http://localhost:7090/mcp

🤖 Claude Desktop

1
编辑 ~/.cursor/mcp.json（或 claude_desktop_config.json）
2
添加 RelayCraft MCP Server 配置
3
Claude 就能读取流量、创建规则、诊断请求了
"relaycraft": { "url": "http://localhost:7090/mcp" }
