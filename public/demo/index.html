RelayCraft
快速上手

一款本地网络调试工具，支持流量捕获、规则重写、AI 辅助、
以及连接 Cursor / Claude 等 AI 编程工具。

HTTP / HTTPS WebSocket 断点调试 AI 建规则 MCP Python 脚本
Step 1

快速上手

安装 → 启动代理 → 安装证书 → 开始抓包

1
下载安装 → GitHub Releases 下载对应平台版本，macOS / Windows / Linux
2
启动应用 → 首次运行会在本机默认端口（127.0.0.1:8080）启动 HTTP 代理
3
安装 CA 证书 → 按指引导出并信任证书，才能抓 HTTPS流量（只需一次）
4
配置系统代理 → 浏览器 / 系统代理指向 127.0.0.1:8080，或使用浏览器插件
5
开始抓包 → Traffic 面板实时显示所有请求，点击查看详情

Step 5

MCP 连接外部 AI

把 RelayCraft 的流量数据暴露给 Cursor / Claude Desktop

🔗 Cursor

1
打开 Cursor → Settings → Models > MCP Servers
2
添加 MCP Server，填入 RelayCraft 提供的地址
3
在 Cursor 对话框里直接问："帮我看看最近的 API 请求有没有问题"
http://localhost:7890/mcp

🤖 Claude Desktop

1
编辑 ~/.cursor/mcp.json（或 claude_desktop_config.json）
2
添加 RelayCraft MCP Server 配置
3
Claude 就能读取你的流量、创建规则、诊断请求了
"mcpServers": { "relaycraft": { "url": "http://localhost:7890/mcp" } }
Step 6

常见使用场景

这几个场景日常开发中最常用

🔄

前后端联调

后端接口未就绪时，用 Map Remote 把请求转发到本地服务，或者 Map Local 返回 mock 数据，前端不受依赖

Map Remote / Map Local
🐛

Bug 复现与诊断

拦截请求/响应，实时修改 Header 或 Body 参数，复现边缘场景，AI 一键诊断 401/500 等错误根因

Breakpoints / AI 诊断
🧪

测试环境切换

线上出问题，一键把 api.example.com 转发到 staging.example.com，不需要改代码，不需要重启服务

Map Remote
